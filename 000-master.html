Latent Growth Mixture Model Plots

Doug Tommet

2017-01-20

Overview

After fitting a growth mixture model, we are often interested in the shape of the trajectories for the latent classes. This presentation shows how to create a figure that has the estimated tractery for each latent class. A similar figure is seen in van de Schoot (2016) (Figure 3). The figure is created in R using ggplot2.

van de Schoot (2016): Figure 3

Overview

This presentation will cover:

  • Creating the data using Mplus monte carlo
  • Reading the data into R
  • Doing a quick exploratory analysis
  • Fitting a growth mixture model in Mplus
  • Using the results from the Mplus analysis to create a plot of trajectories by class.

# http://www.cookbook-r.com/Graphs/Multiple_graphs_on_one_page_(ggplot2)/
# Multiple plot function
#
# ggplot objects can be passed in ..., or to plotlist (as a list of ggplot objects)
# - cols:   Number of columns in layout
# - layout: A matrix specifying the layout. If present, 'cols' is ignored.
#
# If the layout is something like matrix(c(1,2,3,3), nrow=2, byrow=TRUE),
# then plot 1 will go in the upper left, 2 will go in the upper right, and
# 3 will go all the way across the bottom.
#
multiplot <- function(..., plotlist=NULL, file, cols=1, layout=NULL) {
  library(grid)

  # Make a list from the ... arguments and plotlist
  plots <- c(list(...), plotlist)

  numPlots = length(plots)

  # If layout is NULL, then use 'cols' to determine layout
  if (is.null(layout)) {
    # Make the panel
    # ncol: Number of columns of plots
    # nrow: Number of rows needed, calculated from # of cols
    layout <- matrix(seq(1, cols * ceiling(numPlots/cols)),
                    ncol = cols, nrow = ceiling(numPlots/cols))
  }

 if (numPlots==1) {
    print(plots[[1]])

  } else {
    # Set up the page
    grid.newpage()
    pushViewport(viewport(layout = grid.layout(nrow(layout), ncol(layout))))

    # Make each plot, in the correct location
    for (i in 1:numPlots) {
      # Get the i,j matrix positions of the regions that contain this subplot
      matchidx <- as.data.frame(which(layout == i, arr.ind = TRUE))

      print(plots[[i]], vp = viewport(layout.pos.row = matchidx$row,
                                      layout.pos.col = matchidx$col))
    }
  }
}

Create Data

Since we don’t have original data to use in this example, we are going to simulate some data using Mplus’ Monte Carlo option.

I use the R package MplusAutomation to run Mplus through R. The function allows you to pass through most of Mplus’ options. However, one of the few options it doesn’t support is the Monte Carlo option.

To get around this, I created a standard Mplus input file in a text editor. I run this file in Mplus with the runModels() command.

Read in the monte carlo data

If we have actual data to use, we could jump directly to this step and read in the data in the usual way.

MplusAutomation has a nice command to read in data created by Mplus. Otherwise you would need to do a command similar to the one commented out.

The growth mixture model in Mplus likes data in a “wide” format, where each measurement time is a new variable. The package ggplot likes data in a “long” format, where the measurements are stacked in one variable and another variable indicates when the measurement was taken.

This code takes the “wide” data file and converts it to “long”.

data <- getSavedata_Data("monte-carlo.out")

Fit the growth mixture model in Mplus

This code fits a four class growth mixture model. The model assumes a linear trajectory with equally spaced time intervals.

gmm.model.1 <- " %OVERALL% 
                 i s | y1@0 y2@1 y3@2 y4@3 y5@4 y6@5 ;
               "
gmm.body <- mplusObject(
  ANALYSIS = "type=mixture; starts = 100 20;",
  VARIABLE = "classes = c(4); idvariable = id;",
  MODEL = gmm.model.1,
  OUTPUT = "",
  SAVEDATA = "save = cprob; file=model1prob.dat;",
  usevariables = c("id", "y1", "y2", "y3", "y4", "y5", "y6"),
  rdata = data
)
gmm.fit <- mplusModeler(gmm.body, modelout = "gmm001.inp", run = TRUE)

Plot the trajectories of the latent classes

This is my current working version of the code to create the plot. There were many other ways I tried, but all of them failed in some way. In the future, as I learn more about ggplot, I would like to revise this code.

The things I would like to change are:

  • Currently, the parameter estimates need to be manually entered into the plotting function. This breaks the rules of reproducibility. Every time we change the model, the paraameter estimates would need to be reentered.
  • Get the ggplot functions facet_grid and stat_function to work nicely together. Then we can avoid creating individual plots for each class and then combining them.

Since we’re going to be creating the same plot multiple times, we can put it in a function so that it would be easier to edit in the future.

# Creating my own function to do the individual plots.
# a is the class number
# f is the corresponding function to be plotted
myplot <- function(a, f) {
  data.long %>%
  filter(subsample <.25) %>%
  filter(model1.c==a) %>%
    ggplot(aes(x=time, y=y, group=id)) +
      scale_y_continuous(limits = c(0, 30)) +
      geom_line() + 
      stat_function(fun = f, color="blue", size=2) +
      ggtitle(paste("class", a))
  
}
plot1 <- myplot(1, fun.1)
plot2 <- myplot(2, fun.2)
plot3 <- myplot(3, fun.3)
plot4 <- myplot(4, fun.4)
